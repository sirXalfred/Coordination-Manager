# ai-feedback-loop

## Purpose

Processes structured AI feedback entries exported from the Coordination Manager
`/feedback?tab=ai` page. Each entry contains the original user prompt, the AI answer
that was rated, a 2D sentiment score (valence + trust), and optional written feedback.
Produces concrete improvements to AI agent behaviour, system prompts, and documentation.

## When to Use

- User pastes an AI feedback prompt (generated by the "Copy Improvement Prompt" button)
- User asks to review or triage a batch of AI feedback entries
- User wants to improve AI agent quality based on real user sentiment data
- User asks why a specific AI answer scored poorly and how to fix it

## Workflow

1. Parse the feedback block from the pasted prompt -- extract all fields (ID, timestamp, status, valence, trust, prompt, answer, written feedback)
2. Classify feedback signal using sentiment axes:
   - Valence < -0.3 AND Trust < -0.3: CRITICAL (bad + distrusted)
   - Valence < -0.3 AND Trust >= -0.3: HIGH (bad but trusted source)
   - Valence >= -0.3 AND Trust < -0.3: MEDIUM (OK but distrusted)
   - Valence >= 0.3 AND Trust >= 0.3: LOW (reinforce what worked)
   - Otherwise: REVIEW (ambiguous)
3. Diagnose root cause by examining the AI answer against user prompt:
   - Accuracy (factually correct about the platform?)
   - Relevance (answered the actual question?)
   - Completeness (covered all aspects?)
   - Tone (appropriate for context?)
   - Hallucination (invented non-existent features?)
4. Determine improvement targets (one or more):
   - AI system prompt (`apps/api/src/services/ai-chat.ts`)
   - Page context injection (`apps/web/src/pages/*.tsx` setPageContext calls)
   - Documentation (`docs/public/USER_GUIDE.md`, skill files)
   - UI copy/help text (component tooltips, labels)
   - API response messages (`apps/api/src/routes/*.ts`)
5. Produce improvement plan with: Priority, Root Cause, What Went Wrong, Proposed Changes (file + what + why), Verification steps
6. For positive feedback (LOW priority): document what worked well, capture patterns to reinforce in guidelines

## Outputs

| Output | Location | Format |
|--------|----------|--------|
| Feedback analysis | Chat response | Markdown with priority/root cause/changes |
| File edits | Target files from step 4 | Code changes following project conventions |
| Pattern notes | Skill docs or CLAUDE.md | Reinforcement of successful patterns |
| Triage summary | Chat response (batch mode) | Table of entries with priority + root cause |

## Constraints

- NEVER fabricate feedback data or sentiment scores
- NEVER modify the feedback database -- this skill is read-only analysis
- Changes to AI system prompts must preserve existing correct behaviour
- All file edits must follow project conventions (see `.claude/rules/conventions.md`)
- Prefer minimal, targeted fixes over broad rewrites
- When multiple feedback entries point to the same issue, batch into one fix
- ASCII-safe text only in all source files (no smart quotes or em dashes)

## Self-Validation

### Trigger Indicators
- [ ] User pasted a "COORDINATION MANAGER -- AI FEEDBACK IMPROVEMENT REQUEST" block
- [ ] User asked to review AI feedback entries or improve AI quality
- [ ] User referenced the `/feedback?tab=ai` page or sentiment scores

### Completion Markers
- [ ] Feedback classified by priority using valence/trust matrix
- [ ] Root cause diagnosed (accuracy/relevance/completeness/tone/hallucination)
- [ ] At least one improvement target identified with specific file path
- [ ] Improvement plan produced with verification steps
- [ ] File edits follow project conventions (ASCII-safe, no smart quotes)

### Quality Signals
- [ ] Priority matches the severity of the feedback (CRITICAL for bad+distrusted)
- [ ] Root cause matches the actual gap between prompt and answer
- [ ] Proposed changes are minimal and targeted (not broad rewrites)
- [ ] Positive feedback captured as reinforcement patterns
